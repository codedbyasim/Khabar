1. Introduction
1.1 Project Name & Tagline
KHABAR — an Urdu word meaning 'news' or 'awareness' — is an Agentic AI system that transforms raw crisis signals from Pakistan's urban environments into coordinated, intelligent response actions. Google Antigravity is the core orchestration engine for all agent reasoning and tool execution.
1.2 Problem Statement
Pakistan's cities face frequent localized crises: urban flooding, road accidents, heatwaves, building collapses, and infrastructure failures. Current response systems are:
Fragmented across multiple government departments with no unified coordination
Reactive rather than proactive — response begins only after escalation through bureaucratic channels
Slow to process informal crisis signals in Urdu, Punjabi, and Roman Urdu
Unable to convert social media noise into structured, actionable intelligence in real time
Critical signals already exist — on social media, weather APIs, and Google Maps — but they are never automatically converted into coordinated action. KHABAR bridges this gap.

KHABAR uses four coordinated agents within Google Antigravity. Each agent has a specific role, defined inputs, and structured outputs passed to the next agent in the chain.

3.3 The 7 Agentic Tool Functions
The Execution Agent calls these tools via Antigravity's tool execution framework. Every call is logged and visible in the agent trace.

4. Functional Requirements
4.1 Input Processing
FR-01: System shall accept text input in English, Urdu, Roman Urdu, and Punjabi
FR-02: System shall accept photo uploads and assess damage via Gemini Vision API
FR-03: System shall accept voice recordings and transcribe them using Gemini Speech
FR-04: System shall ingest simulated weather API data (rainfall alerts, temperature) every 60 seconds
FR-05: System shall ingest Google Maps traffic congestion data every 60 seconds
FR-06: System shall handle noisy, informal language including abbreviations and slang
4.2 Crisis Detection & Priority Classification
FR-07: Detection Agent shall classify crisis type: flood, fire, collapse, accident, medical, heatwave, or other
FR-08: Detection Agent shall extract location name and attempt GPS geocoding via Google Maps
FR-09: Detection Agent shall output a confidence score (0-100%) for every detection
FR-10: Every detected incident shall be assigned a P1-P5 priority level (see table below)
FR-11: P1 incidents shall immediately trigger the full response pipeline without waiting for human confirmation
Level

4.3 Analysis & Planning
FR-12: Analysis Agent shall estimate number of affected people and vehicles
FR-13: Analysis Agent shall identify nearby critical infrastructure via Google Maps API
FR-14: Analysis Agent shall provide a human-readable impact explanation in Urdu and English
FR-15: Planning Agent shall generate an ordered list of response actions
FR-16: Planning Agent shall check current resource availability before recommending dispatch
FR-17: All recommended actions shall be realistic, domain-relevant, and tied to real Pakistan response agencies
4.4 Action Simulation (Critical Requirement)
FR-18: At minimum one action shall be fully simulated end-to-end for every incident
FR-19: Rescue dispatch simulation shall create a Firestore ticket with team ID, ETA, and incident reference
FR-20: Traffic rerouting simulation shall update a mock routing record and capture old route vs new route
FR-21: Alert broadcast simulation shall generate Urdu message text and log simulated delivery count
FR-22: All simulated actions shall produce timestamped execution logs visible in the dashboard
4.5 Dashboard & Visualization
FR-23: Dashboard shall display explicit before vs after state for each executed action
FR-24: Dashboard shall show the full P1-P5 priority queue with live status indicators
FR-25: Dashboard shall render the complete Antigravity agent reasoning trace per incident
FR-26: Dashboard shall auto-refresh every 5 seconds using Firestore real-time listeners
FR-27: Flutter mobile app shall support incident reporting, dashboard view, and alert notifications

9. Risks, Issues & Mitigation Plan
The following issues are identified before development begins. Each is rated Critical, High, or Medium. All must be addressed in Day 1-2 planning.
Issue
Priority
Detail
Mitigation
What exactly is Google Antigravity?
CRITICAL
Challenge mandates Antigravity (25% score) but exact SDK, APIs, and setup are unclear. May be Google ADK, Vertex AI Agent Builder, or a new platform.
Day 1 priority: Read all Antigravity docs from Google Cloud credits portal. Ask organizers directly in Discord/WhatsApp.
7-day timeline is extremely tight
CRITICAL
May 20 deadline = 7 working days for a full-stack AI + mobile system. Every lost day is critical.
Scope to UC-01 only for the demo. UC-02 and UC-03 are optional extras. Perfect one flow rather than attempt three.
Flutter app is mandatory
HIGH
Challenge requires a working mobile app. If no team member has Flutter experience this is a major risk.
Assign dedicated Flutter developer. Use Firebase + Flutter starter template. Minimum: 3 screens — report, dashboard, alert.
Simulation must be clearly visible to judges
HIGH
Common hackathon failure — judges want to SEE before/after state change, not just hear about it.
Build a dedicated Simulation Log screen. Show: old state, action name, new state, timestamp. Make it the focal point of the demo video.
Loss of offline USP from original KHABAR
HIGH
Original KHABAR's core strength was offline operation. Moving to cloud removes this differentiator entirely.
Reframe the narrative: KHABAR now handles city-scale crises with real-time coordination. Emphasise Urdu support and P1-P5 system as the Pakistan-specific differentiator.
Urdu rendering in Flutter
MEDIUM
Urdu is right-to-left. Flutter requires flutter_localizations and specific font setup for correct RTL Urdu rendering.
Test Urdu rendering on Day 2. Use flutter_localizations package. Fallback: display Urdu in romanized form if full RTL fails.
Mock data must feel realistic
MEDIUM
Generic mock data (City A, Region B) will look amateur to Pakistani judges who know local area names.
Pre-load 5 scenarios with real Islamabad/Lahore area names (G-10, F-7, DHA Phase 5, Gulberg, Saddar). Include Urdu text matching real social media language.
Google Maps API billing
MEDIUM
Maps API calls can accumulate cost during development and live demo, potentially exhausting credits.
Set billing alerts on Day 1. Use Maps Static API for dashboard tiles (cheaper). Cache geocoding results.
